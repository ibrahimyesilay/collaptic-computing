# Informational Collapse

*The defining mechanism of Collaptics: a distributed probability field converging into a single emergent solution.*

```
Classical:  Input → Instructions → Output
Collaptics:        Probability Field → Resonance → Attractor → Collapse → Emergent Solution
```

The system begins in a highly distributed, high-entropy state — millions or billions of candidate solutions coexist. Through resonance and informational gravity, the field reorganizes and eventually **collapses** toward one dominant configuration. That collapse is the answer.

> The machine does not compute. It converges. It does not execute logic; it settles.

## The one equation behind it

Collapse is the evolution of a single stochastic flow — the **Collapse Flow**:

```
dx/dt  =  −Ω²·∇V(x)  +  √(2D)·ξ(t)
```

The field rolls downhill along `−∇V` at a rate set by the coherent resonance `Ω²`, while noise `ξ` keeps it exploring. As coherence rises (or noise falls), the field sharpens onto the deepest attractor. Full derivation: [`../mathematics/collapse-law.md`](../mathematics/collapse-law.md).

## Noise is a feature

In binary computing, noise is error. In Collaptics, the noise term `√(2D)·ξ` is the **search mechanism** — it lets the field escape shallow traps and explore. The ratio `Ω²/D` behaves exactly as an inverse temperature: turning up coherence is the same as cooling the field (confirmed: test P2).

## When collapse fails

Collapse reaches the right answer only when the field can cross the barriers between basins. When barriers are high, the field is **kinetically trapped** and settles into the wrong well — time-to-collapse grows as `exp(βΔV)` (confirmed: tests P5b, P6). This is the central limit of the whole framework: an energy/throughput advantage on smooth, crossable landscapes, not a universal solver.
